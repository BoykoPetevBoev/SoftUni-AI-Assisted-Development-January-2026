Why Developers Like VS Code

VS Code is popular because it scales from quick edits to full projects. You can start with a simple folder, then add extensions for your language, formatter, linter, test runner, containers, or remote development.

  • Productive editing: multi-cursor, find/replace, snippets, and powerful navigation.
  • Language features: IntelliSense, symbol search, rename, go-to definition, diagnostics.
  • Integrated workflows: Git, terminal, tasks, debugging, and tests from one place.
  • Customizable: settings sync, themes, keybindings, and workspace-specific settings.

Core Areas

  • Explorer: browse and manage files.
  • Editor: tabs, split views, code actions.
  • Source Control: stage/commit, diffs, branches.
  • Run & Debug: breakpoints, watch, call stack.
  • Extensions: add language servers, tools, themes.

Tip: Keep a project in a single folder and use a workspace file only when you need multi-root setups.

Extensions Worth Knowing

VS Code ships with strong defaults, but extensions unlock language servers, formatting, linting, and tooling.

  • Language support: Python, C#, Java, Go, Rust, etc.
  • Formatting: Prettier (web), Black (Python), etc.
  • Linting: ESLint (JS/TS), Pylint/Ruff (Python), etc.
  • Remote: SSH, WSL, and Containers for consistent dev environments.

Good practice: enable format-on-save and keep project settings in a repository (workspace settings).

Git in VS Code (in one minute)

The Source Control view shows file changes, lets you stage selected hunks, write commit messages, and view diffs. For advanced workflows, you can still use the integrated terminal.

  • Stage changes, commit, and push without leaving the editor.
  • Use the timeline and inline blame (with Git extensions) to understand history.
  • Review pull requests with extensions when you need code review inside VS Code.
